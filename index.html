Java Database Connectivity (JDBC) Documentation

A comprehensive guide to connecting Java applications with relational databases

What is JDBC?

JDBC (Java Database Connectivity) is an API in Java that enables applications to connect to relational databases, execute SQL queries, and retrieve or manipulate data. It serves as a bridge between Java code and the database.

Key Point: JDBC provides a standard interface for Java applications to interact with various databases, allowing developers to write database-agnostic code.

JDBC Workflow Overview

1

Load the JDBC Driver

Register the appropriate database driver with the DriverManager.

2

Establish a Connection to the Database

Create a connection using the DriverManager with database URL, username, and password.

3

Create and Execute SQL Statements

Use Statement or PreparedStatement objects to execute SQL queries.

4

Process Returned Results

Handle the ResultSet object to retrieve and process query results.

5

Close the Connection

Properly close database resources to prevent memory leaks.

Project Setup

Add JDBC Driver (Example for MySQL/MariaDB using Maven):

<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <version>8.3.0</version>
</dependency>

For Gradle users:

implementation 'mysql:mysql-connector-java:8.3.0'

Basic JDBC Example

import java.sql.*;

public class Main {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/schooldb";
        String username = "root";
        String password = "your_password";

        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
            Connection con = DriverManager.getConnection(url, username, password);
            Statement stmt = con.createStatement();
            ResultSet rs = stmt.executeQuery("SELECT * FROM students");

            while (rs.next()) {
                System.out.println(rs.getInt("id") + "  " + rs.getString("name"));
            }

            con.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Using PreparedStatement

PreparedStatement provides better performance and security by preventing SQL injection:

String sql = "INSERT INTO students(name, age) VALUES (?, ?)";
PreparedStatement ps = con.prepareStatement(sql);
ps.setString(1, "Denis");
ps.setInt(2, 21);
ps.executeUpdate();

Common JDBC Classes

  • DriverManager: Manages database drivers and connections
  • Connection: Represents an active database connection
  • Statement: Executes SQL queries
  • PreparedStatement: Executes parameterized SQL queries
  • ResultSet: Stores and provides access to returned data
  • CallableStatement: Executes stored procedures
  • DatabaseMetaData: Provides database information

CRUD Operations

Examples for Create, Read, Update, Delete using JDBC.

Create

String sql = "INSERT INTO users(name,email) VALUES(?,?)";
PreparedStatement ps = con.prepareStatement(sql);
ps.setString(1, "John");
ps.setString(2, "john@email.com");
ps.executeUpdate();

Read

ResultSet rs = stmt.executeQuery("SELECT * FROM users");
while (rs.next()) {
    System.out.println(rs.getString("name"));
}

Update

String sql = "UPDATE users SET name=? WHERE id=?";
PreparedStatement ps = con.prepareStatement(sql);
ps.setString(1, "New Name");
ps.setInt(2, 1);
ps.executeUpdate();

Delete

String sql = "DELETE FROM users WHERE id=?";
PreparedStatement ps = con.prepareStatement(sql);
ps.setInt(1, 1);
ps.executeUpdate();

Closing Connections

Always close connections to avoid memory leaks:

// Close resources in reverse order of creation
if (rs != null) rs.close();
if (stmt != null) stmt.close();
if (con != null) con.close();

Best Practice: Use try-with-resources for automatic resource management:

try (Connection con = DriverManager.getConnection(url, username, password);
     PreparedStatement ps = con.prepareStatement(sql)) {
    // Your database operations here
} catch (SQLException e) {
    e.printStackTrace();
}

Conclusion

JDBC is a powerful API that forms the foundation for interacting with relational databases in Java. MariaDB can be used as a replacement for MySQL on Kali Linux, fully compatible with JDBC.

By following the patterns and best practices outlined in this documentation, you can build robust, efficient database applications in Java.
